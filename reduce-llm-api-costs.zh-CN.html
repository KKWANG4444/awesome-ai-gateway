如何降低你的 LLM API 成本

账单由少数几个杠杆主导。下面按影响力大致排序——先拉最上面的几个;相比之下,靠后的那些只是零头。

独立 · 无厂商返利 · 属于 Awesome AI Gateway 评测 · 价格变化快——请在实时数据上计算,别信静态榜单。

  1. 最大杠杆
    选对模型,别默认旗舰同一任务在不同模型间能差 100×+。用能过你质量评测的最便宜模型——这一个决定通常压倒下面一切。按你的负载估算 →
  2. 高
    砍输出 token输出(completion)token 通常比输入贵好几倍。约束 max_tokens、要求简洁、不需要时别让它"解释推理过程"。
  3. 高(若你复用上下文)
    缓存并精简输入对大段稳定前缀(系统提示、长上下文)用提示缓存——许多厂商对缓存输入大幅折扣。优先用检索,而非把整个语料每次都塞进调用。
  4. 中
    按任务难度路由简单请求发给便宜模型,只有难的才升级(级联 / 路由器)。有网关时这只是改配置,而非重写。
  5. 规模化时中等
    避开按 token 加价托管聚合器的充值费在高用量下会累积——BYOK 控制面或自托管网关能去掉它。(仍小于模型选择。)
  6. 取决于负载
    批处理、转异步对非交互式任务,批处理 API 往往比实时调用有可观折扣。
⚖️ 注意这个排序:选哪个模型胜过所有网关/基础设施技巧的总和。先优化它,再用缓存、路由和预算把其余的工程化。从这里开始 → 成本计算器。

那"最便宜的 LLM API"到底是哪个?

没有恒定答案——价格按月变动,重要的是"对你的任务够用的最便宜模型",而非绝对地板价。所以本项目提供一个基于评测同一份开放价格数据的计算器,而不是一个下周就过时的数字。填入你真实的输入/输出 token 组合来对比。

网关在其中的位置

网关是你把这些杠杆工程化到全团队的方式——响应缓存、模型路由、按 key 的预算与限流集中在一处。用选网关工具按需挑一个;只是记住,模型仍主导账单。

常见问题

最便宜的 LLM API 是哪个?

一直在变——信实时价格。能过你质量关的最便宜模型(而非绝对最便宜)才是答案;按你自己的 token 组合计算。

输入还是输出 token 更贵?

输出通常更贵——常贵好几倍。砍输出比砍 prompt 更快压低账单。

提示缓存能省钱吗?

复用大段稳定前缀时能——许多厂商对缓存输入大幅折扣。

AI 网关能降本吗?

间接能——缓存、路由、预算,以及 BYOK/自托管避开加价。但模型选择主导。
